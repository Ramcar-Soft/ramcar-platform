**Branch**: `006-fix-desktop-dropdown` | **Date**: 2026-04-09 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/006-fix-desktop-dropdown/spec.md`

## Summary

The desktop app's sidebar avatar dropdown shows a scroll indicator but the menu items (Account, Logout) are invisible. Root cause: the desktop app runs React 18.3.1 while the shared `@ramcar/ui` shadcn components target React 19 (no `forwardRef` on `SidebarMenuButton`). Without proper ref forwarding, Radix Popper can't measure the trigger element, causing `--radix-dropdown-menu-content-available-height` to be near-zero, which constrains the dropdown's visible area via `max-height`. Fix: upgrade desktop React to 19.x to match web.

## Root Cause Analysis

### Why the dropdown shows scroll but invisible items

1. `SidebarMenuButton` (`packages/ui/src/components/ui/sidebar.tsx:498`) is a plain function component (zero `forwardRef` usage in the file ‚Äî confirmed via grep)
2. In the sidebar footer, it's used as: `<DropdownMenuTrigger asChild><SidebarMenuButton size="lg">...</SidebarMenuButton></DropdownMenuTrigger>`
3. Radix's `DropdownMenuTrigger` with `asChild` uses Slot to merge props + ref onto the child
4. **React 18**: Slot's `React.cloneElement()` cannot merge `ref` onto plain function components ‚Üí ref is silently dropped
5. Without the ref, Radix Popper can't measure the trigger ‚Üí `--radix-dropdown-menu-content-available-height` is 0 or near-zero
6. `DropdownMenuContent` has `max-h-(--radix-dropdown-menu-content-available-height)` (`dropdown-menu.tsx:43`) ‚Üí constrains visible area to ~0px
7. Menu items exist (causing scroll) but the viewport is too small to display them
8. **React 19**: `ref` is a regular prop on function components ‚Üí Slot can forward it ‚Üí Popper measures correctly ‚Üí dropdown renders at full height

### Why web works but desktop doesn't

- Web: `react: 19.2.3` ‚Üí ref forwarding works
- Desktop: `react: ^18.2.0` (installed: 18.3.1) ‚Üí ref forwarding fails
- Both use identical `@ramcar/ui` components (same `SidebarMenuButton`, same `DropdownMenu`)

### Evidence

- `packages/ui/src/components/ui/sidebar.tsx` ‚Äî zero `forwardRef` calls (grep confirmed)
- `apps/desktop/package.json:23` ‚Äî `react: ^18.2.0`
- `apps/web/package.json` ‚Äî `react: 19.2.3`
- `@types/react: ^19` already in desktop devDependencies (types say ref is a prop, but React 18 runtime disagrees)
- `pnpm typecheck` passes after upgrade (no type errors)
