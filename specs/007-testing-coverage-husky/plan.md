# Implementation Plan: Testing Infrastructure, Coverage & Pre-Commit Quality Gates

**Branch**: `007-testing-coverage-husky` | **Date**: 2026-04-09 | **Spec**: [spec.md](./spec.md)  
**Input**: Feature specification from `/specs/007-testing-coverage-husky/spec.md`

## Summary

Add comprehensive testing infrastructure to the Ramcar Platform monorepo: Vitest for frontend/package unit testing, preserve Jest for NestJS API, Playwright for web E2E tests, V8/Istanbul coverage reporting across all workspaces, and Husky + lint-staged pre-commit hooks that block commits failing lint or typecheck.

## Technical Context

**Language/Version**: TypeScript 5.x (strict mode), Node.js 22 LTS  
**Primary Dependencies**: Vitest (frontend + packages), Jest + ts-jest (api), Playwright, Husky v9, lint-staged  
**Storage**: N/A (no data persistence — development tooling only)  
**Testing**: Vitest (web, www, desktop, shared, store, ui), Jest (api), Playwright (web E2E)  
**Target Platform**: macOS/Linux development environments, CI runners  
**Project Type**: Monorepo development tooling (cross-workspace)  
**Performance Goals**: Pre-commit hooks complete within 60 seconds; unit test suites execute within 30 seconds per workspace  
**Constraints**: Must not break existing Jest setup in apps/api; must work with pnpm workspaces + Turborepo  
**Scale/Scope**: 4 apps (web, www, api, desktop) + 6 packages (config, db-types, i18n, shared, store, ui)

## Implementation Phases

### Phase 1: Shared Vitest Config + packages/config Setup

**Goal**: Create the shared Vitest base configuration and test setup files in packages/config.

**Files to create/modify**:
- `packages/config/vitest.config.ts` — shared base config (v8 coverage, text + lcov reporters, include/exclude patterns)
- `packages/config/vitest.setup.ts` — shared setup (import @testing-library/jest-dom/vitest)
- `packages/config/package.json` — add vitest config export, add devDependencies (vitest, @testing-library/jest-dom)
- `packages/config/tsconfig.node.json` — verify vitest types are included

**Dependencies**: None (first phase)

### Phase 2: Vitest in Each Frontend Workspace + Shared Packages

**Goal**: Add Vitest configuration and test scripts to each workspace that doesn't have testing yet.

**Workspaces to configure**:
- `apps/web` — vitest.config.ts (jsdom env, extends shared), package.json scripts (test, test:watch, test:cov)
- `apps/www` — vitest.config.ts (jsdom env), package.json scripts
- `apps/desktop` — vitest.config.ts (jsdom env), package.json scripts
- `packages/shared` — vitest.config.ts (node env), package.json scripts
- `packages/store` — vitest.config.ts (jsdom env), package.json scripts
- `packages/ui` — vitest.config.ts (jsdom env), package.json scripts

**Per workspace**:
- Add devDependencies: vitest, @testing-library/react, @testing-library/jest-dom, @testing-library/user-event (React workspaces only)
- Add test scripts: `test`, `test:watch`, `test:cov`
- Create vitest.config.ts extending the shared config

**Dependencies**: Phase 1 (shared config must exist)

### Phase 3: Jest Coverage Enhancement for apps/api

**Goal**: Ensure apps/api Jest config generates coverage in the same format as Vitest workspaces.

**Files to modify**:
- `apps/api/package.json` — verify `test:cov` script exists (already has `jest --coverage`), ensure lcov reporter is configured

**Dependencies**: None (can run in parallel with Phase 2)

### Phase 4: Playwright E2E Setup for apps/web

**Goal**: Add Playwright end-to-end testing infrastructure to apps/web.

**Files to create/modify**:
- `apps/web/e2e/playwright.config.ts` — Playwright config (webServer auto-start, screenshot on failure, trace on first retry, HTML reporter)
- `apps/web/package.json` — add devDependencies (@playwright/test), add scripts (test:e2e, test:e2e:ui, test:e2e:report)
- `apps/web/.gitignore` — add playwright-report/, test-results/

**Dependencies**: None (independent from unit test setup)

### Phase 5: Husky + lint-staged Pre-Commit Hooks

**Goal**: Add pre-commit hooks that block commits on lint or typecheck failures.

**Files to create/modify**:
- Root `package.json` — add `prepare` script (`husky`), add devDependencies (husky, lint-staged), add lint-staged config
- `.husky/pre-commit` — shell script that runs lint-staged + turbo typecheck
- `.lintstagedrc.json` or inline config — run ESLint on staged `*.ts` and `*.tsx` files

**lint-staged config**:
```json
{
  "*.{ts,tsx}": ["eslint --max-warnings 0"]
}
```

**Pre-commit script**:
```bash
#!/bin/sh
npx lint-staged
pnpm typecheck
```

**Dependencies**: None (independent from test setup)

### Phase 6: Turbo Pipeline + Root Scripts Update

**Goal**: Ensure Turbo's task pipeline handles test:e2e and test:cov correctly, and root scripts are complete.

**Files to modify**:
- `turbo.json` — add `test:e2e` task (no cache, persistent), verify `test` task configuration
- Root `package.json` — add `test:e2e` and `test:cov` scripts if needed

**Dependencies**: Phases 2, 4, 5 complete

### Phase 7: Smoke Tests + Verification

**Goal**: Add one example unit test per workspace and one example E2E test to verify the entire setup works.

**Files to create**:
- `packages/shared/src/validators/__tests__/example.test.ts` — simple Zod schema validation test
- `apps/web/src/app/page.test.tsx` — basic render test (if applicable)
- `apps/web/e2e/smoke.spec.ts` — Playwright test that loads the app and checks basic rendering
- Verify `pnpm test` runs all unit tests
- Verify `pnpm --filter @ramcar/web test:e2e` runs Playwright
- Verify pre-commit hook blocks a bad commit

**Dependencies**: All previous phases

### Phase 8: ESLint Config Update for Test Files

**Goal**: Ensure ESLint is configured to recognize test globals and test-specific patterns.

**Files to modify**:
- `packages/config/eslint.config.mjs` — add overrides for test files (*.test.ts, *.test.tsx, *.spec.ts) to allow test globals and devDependency imports
- Verify no lint errors in test files

**Dependencies**: Phase 2 (test files must exist to verify)

### Phase 9: Update .gitignore and CLAUDE.md

**Goal**: Ensure all generated artifacts are gitignored and documentation is updated.

**Files to modify**:
- Root `.gitignore` — verify `coverage/`, add `playwright-report/`, `test-results/`, `blob-report/`
- `CLAUDE.md` — update Active Technologies and Recent Changes sections with testing infrastructure details

**Dependencies**: All previous phases
