# dotenvx Integration Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Integrate dotenvx into the monorepo to encrypt `.env.local`, `.env.development`, and `.env` (production) files, commit them to git, and wrap all Turbo scripts with `dotenvx run`.

**Architecture:** dotenvx wraps Turborepo at the root `package.json` level. Three encrypted env files (local, development, production) are committed to git. Decryption keys live in `.env.keys` (gitignored). All apps inherit env vars via process environment inheritance.

**Tech Stack:** dotenvx, pnpm, Turborepo, git

**Spec:** `docs/superpowers/specs/2026-03-16-dotenvx-integration-design.md`

---

## File Map

| Action | File | Responsibility |
|--------|------|---------------|
| Create | `.env.local` | Localhost env vars (local Supabase, local API) |
| Create | `.env.development` | Remote dev env vars (remote Supabase dev project) |
| Create | `.env` | Production env vars |
| Create | `.env.example` | Plaintext template with placeholder values |
| Modify | `.gitignore` | Remove old env rules, add `.env.keys` |
| Modify | `turbo.json` | Add missing vars to `globalEnv` |
| Modify | `package.json` | Wrap scripts with `dotenvx run`, add dotenvx devDependency |
| Generated | `.env.keys` | Auto-generated by `dotenvx encrypt` (gitignored) |

### Task 3: Encrypt env files with dotenvx

**Files:**
- Modify: `.env.local` (encrypted in-place)
- Modify: `.env.development` (encrypted in-place)
- Modify: `.env` (encrypted in-place)
- Generated: `.env.keys`

- [ ] **Step 1: Encrypt all three env files**

Run:
```bash
pnpm dotenvx encrypt -f .env.local
pnpm dotenvx encrypt -f .env.development
pnpm dotenvx encrypt -f .env
```

Expected: Each command outputs a message like `encrypted .env.local`. The `.env.keys` file is created (or appended to) with `DOTENV_PRIVATE_KEY_LOCAL`, `DOTENV_PRIVATE_KEY_DEVELOPMENT`, and `DOTENV_PRIVATE_KEY_PRODUCTION`.

- [ ] **Step 2: Verify encryption worked**

Run:
```bash
head -5 .env.local
```

Expected: The file should contain encrypted content starting with comments like `#/-------------------[DOTENV_PUBLIC_KEY_LOCAL]-------------------/` followed by encrypted values.

- [ ] **Step 3: Verify decryption works**

Run:
```bash
pnpm dotenvx run -f .env.local -- printenv | grep SUPABASE_URL
```

Expected: prints `SUPABASE_URL=http://127.0.0.1:54321` (the decrypted value).

Note: If encryption fails or produces unexpected results, restore from the backup files: `cp .env.backup .env && cp .env.development.backup .env.development`.

- [ ] **Step 4: Back up `.env.keys` securely**

**MANUAL STEP:** Copy `.env.keys` to a secure vault (1Password, etc.). This file contains the decryption keys for all environments. If lost, you lose access to the encrypted secrets.

- [ ] **Step 5: Delete backup files**

Run:
```bash
rm .env.backup .env.development.backup
```
